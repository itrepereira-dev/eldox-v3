O Eldox supera o AutoDoc em inteligência (IA), aprovações, integração de módulos e abrangência de produto.

🚨 Gap Crítico — App Mobile

📱 Mobile com Offline — Decisão Estratégica

CRÍTICO PARA LANÇAMENTO

O AutoDoc tem um app Android nativo que permite inspeção totalmente offline. O inspetor de campo vai à obra, faz inspeções sem internet (modo avião), e quando volta ao wifi/4G, o app sincroniza automaticamente. Sem isso, o inspetor precisa de conexão para cada clique — inviável em obras com sinal ruim.

Impacto: ALTO Afeta: Adoção por inspetores de campo Diferencial atual AutoDoc: app nativo Android

Caminhos para resolver (em ordem de custo/tempo):


Opção A — PWA com Service Worker (RECOMENDADO para lançamento rápido):
Eldox já é React. Adicionar service worker + IndexedDB para fila offline. O inspetor instala o PWA no Android, funciona offline, sincroniza quando há conexão. Tempo estimado: 3-4 semanas de desenvolvimento.


Opção B — React Native (longo prazo):
App nativo para Android (e iOS) com Expo. Experiência superior mas custo maior. Reutiliza componentes React com adaptações. Tempo: 3-4 meses.


Opção C — Lançar sem offline (decisão de negócio):
Se os clientes-alvo têm obras com boa cobertura de internet (obras em SP/grandes cidades), pode lançar como web responsivo e implementar offline v2. Risco: perder para AutoDoc em obras com sinal ruim.

Conclusão: O Eldox pode ser lançado para clientes com obras em áreas de boa cobertura de internet (obras urbanas, regiões metropolitanas) imediatamente — o produto já supera o AutoDoc em inteligência e breadth de funcionalidades. Os outros gaps (nomenclatura, relatório de uso, email) são P2/P3 e podem ir no roadmap pós-lançamento.
