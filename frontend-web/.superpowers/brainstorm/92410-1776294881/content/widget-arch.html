Arquitetura do Widget Generator

Como novos KPIs se tornam widgets automaticamente — sem mexer no core do dashboard.

Para adicionar um widget novo:
1
Criar o componente do widget
// widgets/NcsAbertasWidget.tsx
export function NcsAbertasWidget({ config }) {
  // busca dados, renderiza KPI
}
2
Registrar no WidgetRegistry
registry.register({
  id: 'ncs-abertas',
  titulo: 'NCs Abertas',
  modulo: 'ncs',
  tamanhos: ['1x1', '2x1'],
  configSchema: obraIdOptional,
  component: NcsAbertasWidget,
})
✓
Aparece automaticamente no catálogo
Sem mexer no DashboardGrid, no modal de adicionar widget, nem no sistema de persistência.
O que cada WidgetDefinition descreve:
interface WidgetDefinition {
  id: string // slug único
  titulo: string // label no catálogo
  descricao: string // tooltip
  icone: ReactNode // lucide icon
  modulo: string // ncs|fvs|alm...
  tamanhos: GridSize[] // 1x1 | 2x1 | 2x2 | 4x2
  configSchema: ZodSchema // valida params
  roles: Role[] // quem pode ver
  tier: 1 | 2 | 3 // global|obra|cruzado
  component: React.FC // o widget em si
  preview: string // screenshot estático
}
Layout salvo por usuário (JSON no banco):
[
  { widgetId: 'ncs-abertas',
    x: 0, y: 0, w: 2, h: 1,
    config: { obraId: 3 } },
  { widgetId: 'semaforo-geral',
    x: 2, y: 0, w: 4, h: 2,
    config: {} }
]
